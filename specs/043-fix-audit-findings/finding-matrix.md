# Selected-Finding Traceability Matrix: Feature 043

This is the selected-finding traceability matrix for feature 043, derived from
`tasks.md`, `spec.md`, `plan.md`, `contracts/audit-remediation.md`,
`research.md`, `data-model.md`, and `quickstart.md`. Each row maps a planned
task to its audit or finding source, normative OPC UA section or local
regression rule, expected status or local outcome, required state assertion, and
the targeted command or evidence update that proves the task.

Draft status placeholders are resolved from the referenced OPC UA MCP
section and the per-task verification ledger. Verification-only tasks map to
their inline command and the required `verification.md` evidence update.

## MCP Citation Check

Before implementation starts for any OPC-UA-behavior task, confirm the exact
OPC UA document and section, table, or method with the OPC UA MCP reference and
record that citation in this matrix or in the task's verification entry. Local
setup, helper, documentation, and final verification tasks may cite a local
traceability, helper, or verification rule when they do not change OPC UA
behavior.

Any task whose expected status or section details are still unresolved must
resolve that citation/status from the referenced OPC UA MCP source before
production code changes begin for that task.
