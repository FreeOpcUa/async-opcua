# OPC UA Conformance Audit Findings (Part 5 & Part 8)

## Summary of Candidate Divergences

1. **`LocaleIdArray` under `ServerCapabilities` Returns Null**: The mandatory property `LocaleIdArray` under `ServerCapabilities` is not matched/populated in the core read handler (`core.rs`), falling back to returning a default `Null` value.
2. **`SoftwareCertificates` under `ServerCapabilities` Returns Null**: The mandatory property `SoftwareCertificates` under `ServerCapabilities` is not matched/populated in the core read handler (`core.rs`), falling back to returning a default `Null` value.
3. **`MinSupportedSampleRate` DataType Mismatch**: The specification defines `MinSupportedSampleRate` as having datatype `Duration` (`Double` in OPC UA). However, the implementation in `core.rs:329` casts the value and returns a `u32` (`(min_sampling_interval_ms as u32).into()`).
4. **`NamespaceMetadataType` Properties NodeClass Mismatch**: In `diagnostics/node_manager.rs:427`, reading the `NodeClass` attribute of the namespace metadata property nodes (e.g., `NamespaceUri`, `IsNamespaceSubset`, etc.) returns `NodeClass::Object` instead of `NodeClass::Variable`.
5. **Mandatory `ServerDiagnosticsType` Components Missing**: While session and subscription diagnostics summary counters are implemented, the mandatory components `EnabledFlag` (node `2294`), `SubscriptionDiagnosticsArray` (node `2290`), and `SessionsDiagnosticsSummary` (node `3706`) are missing from the diagnostics implementation.
6. **No `SemanticsChanged` Support or Dynamic `EURange` Updates**: The `SemanticsChanged` StatusCode bit is defined in the types but never set or used by the server when `EURange` or `EngineeringUnits` change. Active monitored items cache `EURange` at construction and never update it if the underlying variable is modified.
7. **Monitored Item modification (`modify_monitored_items`) Does Not Fetch `EURange`**: Modifying a monitored item does not trigger a browse/read of the target node's `EURange`. If an item is modified to use a percent deadband filter but was not originally created with one, the modification fails with `BadDeadbandFilterInvalid` because the cached range is `None`.
