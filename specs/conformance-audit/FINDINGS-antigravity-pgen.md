# OPC UA Conformance Audit: General Service Behaviour (Â§5.1-5.4)

## Summary of Candidate Divergences

1. **Rule 1: Diagnostic Information Population Gap (`returnDiagnostics`)**
   - **Description**: The server does not support actually populating diagnostic information when requested by the client using the `returnDiagnostics` bitmask in `RequestHeader`.
   - **Service-Level**: `ResponseHeader.service_diagnostics` is always hardcoded to `DiagnosticInfo::default()` (empty/all-None) and `ResponseHeader.string_table` is always hardcoded to `None` in `ResponseHeader::new_timestamped_service_result`.
   - **Operation-Level**:
     - In services like `Read`, `Write`, `Call`, and `NodeManagement`, requesting diagnostics allocates/returns arrays of default (empty) `DiagnosticInfo` elements because `.set_diagnostic_info(...)` is never called by node managers or handlers.
     - In all other services (e.g., `HistoryRead`, `HistoryUpdate`, `Browse`, `BrowseNext`, `QueryFirst`, `QueryNext`, `MonitoredItems` services, `Subscriptions` services), the server unconditionally returns `None` for the `diagnostic_infos` field, ignoring `RequestHeader.return_diagnostics`.

2. **Rule 6: Locale Negotiation Incomplete Implementation**
   - **No Attribute Localization**: The session's negotiated `locale_ids` (stored during `ActivateSession` in `Session.locale_ids`) are never read or used by node managers. Thus, reading localized node attributes (e.g., `DisplayName`, `Description`) always returns the static value stored in the node, with no locale negotiation or filtering.
   - **Missing 'mul' / 'qst' Special Locales Handling**: The special `'mul'` and `'qst'` locales are not recognized. If requested during `FindServers`/`GetEndpoints`, `supports_locale_ids` treats them as regular locale tags. Because they don't match the server's configured locales, the request is rejected with an empty endpoint list.
   - **Lack of Write Restriction**: There is no validation or check in `Write` services to reject client attempts to write `LocalizedText` with `'mul'` or `'qst'` locales.
