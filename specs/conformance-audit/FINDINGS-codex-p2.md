# OPC UA Part 2/4 Security-Conformance Findings

## Candidate Divergences

1. OpenSecureChannel accepts/renews a peer certificate after only structural parsing and proof-of-possession verification; the Part 4 §6.1.3 trust-chain validation pipeline is not invoked in the OpenSecureChannel path before the SecureChannel is accepted.
2. SecureChannel `Renew` does not re-run ApplicationInstanceCertificate verification, although Part 4 §6.1.7 says certificate verification shall execute every time the SecurityToken is renewed.
3. Suppressed certificate-validation failures are logged with `warn!` but no OPC UA AuditCertificate event is emitted, although Part 4 §6.1.3 says suppressed validation errors are always reported via auditing.
4. X.509 user-token signatures are verified before any visible §6.1.3 validation of the user signing certificate; authentication later checks only the configured thumbprint.
