# OPC UA Part 5/Part 8 Conformance Findings

## Candidate Divergences Summary

1. `Server_ServerStatus` aggregate direct reads return the cached `ServerStatusDataType` without refreshing `current_time`; only the `CurrentTime` component read and subscription sampler produce live time.
2. `ServerCapabilities.LocaleIdArray` exists in generated namespace nodes but has no runtime value branch in the core server value reader, unlike the other scoped ServerCapabilities values.
3. NamespaceMetadata property reads report `NodeClass::Object` for property nodes even though Part 5 defines NamespaceMetadata members as Variables.
4. NamespaceMetadata `StaticNumericNodeIdRange` is exposed with scalar `ValueRank` even though Part 5 defines `NumericRange[]`.
5. Dynamic namespace metadata objects are generated for namespaces whose mandatory NamespaceMetadata values may be absent/defaulted, despite Part 5 saying clients should not assume namespaces are listed if mandatory properties cannot be filled.
6. `ModifyMonitoredItems` does not fetch `EURange` before parsing a newly requested PercentDeadband filter; only create-time percent deadband requests trigger the `EURange` browse/read path.
