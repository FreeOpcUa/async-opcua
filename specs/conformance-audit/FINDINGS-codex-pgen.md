# OPC UA Part 4 v1.05.07 §5.1-5.4 Conformance Findings

## Candidate Divergences Summary

1. `returnDiagnostics` service-level diagnostics and `stringTable` appear unimplemented globally: `ResponseHeader::new_timestamped_service_result` always initializes `service_diagnostics` to `DiagnosticInfo::default()` and `string_table` to `None`, while `ServiceFault::new` uses that constructor for service failures.
2. `returnDiagnostics` per-operation diagnostics are only implemented in some paths. `Read`, `Write`, `Call`, and node-management services route through diagnostic-aware helpers, but Browse, BrowseNext, TranslateBrowsePathsToNodeIds, HistoryRead, HistoryUpdate, monitored-item services, and DeleteSubscriptions return `diagnostic_infos: None`.
3. Serialized responses larger than the client `maxResponseMessageSize` are not visibly converted to `Bad_ResponseTooLarge`: the session stores the negotiated value, but the send path found in this audit does not reference it and no server-side `BadResponseTooLarge` use was found.
4. Locale negotiation for `FindServers`/`GetEndpoints` appears to filter endpoints by whether any requested locale is supported instead of returning the most-preferred available translation or any available translation on no match.
5. Special `mul`/`qst` locale behavior was not found: no handling of `mul` or `qst` in locale matching/selection, and no Write-specific rejection path for special locales was found.
