# OPC Foundation Security Advisory Review

Reviewed every published advisory in [github.com/OPCFoundation/OPC-SecurityAdvisories](https://github.com/OPCFoundation/OPC-SecurityAdvisories)
(2017–2025) against async-opcua. For each, the question is: *does the underlying
vulnerability class exist in this (Rust) stack?* Memory-safety bugs in the legacy ANSI C /
Java stacks and Windows-deployment issues are inherently N/A to a safe-Rust implementation;
the protocol- and crypto-level ones are verified against our code below.

Date: 2026-06-24. **Result: no default configuration of async-opcua is exposed to any of these.**

## Protocol / crypto level — verified mitigated

| Advisory | Class | async-opcua status | Where |
|---|---|---|---|
| CVE-2018-12086, CVE-2021-27432 | Stack overflow via deeply nested structures | **Mitigated** | `MAX_DECODING_DEPTH=10` + `depth_lock()` on ExtensionObject / DataValue / DiagnosticInfo / Variant (`async-opcua-types/src/encoding.rs` `DepthGauge`) |
| CVE-2020-8867 | Server crash on malformed messages | **Mitigated** | decode returns `Err`, never panics; continuous fuzzing (`fuzz/fuzz_targets/*`) |
| CVE-2022-29863, CVE-2023-27321, CVE-2024-45526 | OOM / unbounded allocation / perf degradation | **Mitigated** | `DecodingOptions` (`max_message_size`, `max_chunk_count`, `max_string_length`, `max_byte_string_length`, `max_array_length`); in-flight request bounding |
| CVE-2022-29862 | Hang during SecureChannel creation | **Mitigated** | `hello_timeout` handshake deadline (tcp / websocket / reverse_tcp) |
| GCVE-105-2025-001 | Connection exhaustion via improper shutdown | **Mitigated** | `max_connections` + `max_connections_per_ip`; `FuturesUnordered` reaps finished tasks → `connection_map.remove`; `catch_unwind` so a panicking task can't leak a slot (`async-opcua-server/src/server.rs`) |
| CVE-2022-29865, CVE-2020-29547 | Rogue/untrusted peer establishes secure connection | **Mitigated** | `trust_client_certs = false` by default; cert-chain validation (feature 013); OpenSecureChannel trust-gap fix (P2-SEC-01) |
| CVE-2018-7559, CVE-2024-42512 | Basic128Rsa15 RSA PKCS#1 v1.5 padding/timing oracle | **Not exposed** | Basic128Rsa15 is not in the default endpoint set and is `is_deprecated()`; default `aws-lc-rs` build does constant-time v1.5 decrypt with implicit rejection. Residual: pure-Rust `rsa`-crate build carries RUSTSEC-2023-0071 (Marvin) — only relevant if a user opts out of aws-lc-rs *and* manually enables the deprecated policy |
| CVE-2018-12087, CVE-2019-19135 | Identity-token password/cert intercepted + reused | **Mitigated** | `server_nonce` bound into identity-token decrypt (`legacy_secret_decrypt`) and `verify_x509_identity_token`; ECC token nonce binding (features 015/016); session-hijack binding (PR #101) |
| CVE-2022-29866, CVE-2024-42513 | HTTPS endpoint: stack overflow / auth bypass | **N/A** | No legacy SOAP/HTTPS endpoint. `opc.wss` reuses the *same* `Transport` + SecureChannel as `opc.tcp` (TLS is only the wrapper; OPC-UA application authentication is unchanged), and the shared decoder is depth-limited |
| GCVE-105-2025-002 | Spec-level (OPC UA Specification) | **Monitor** | Advisory body is "TBA" (reserved/embargoed); revisit when published |

## N/A — platform / language / deployment specific

- CVE-2017-11672 (LDS unquoted Windows service path), CVE-2017-12070 (Windows DLL planting),
  CVE-2017-17443 / CVE-2022-44725 (standalone LDS local-file / privilege) — Windows install &
  the standalone C LDS; our discovery is an in-memory registry, no installed service.
- CVE-2021-27434 (.NET Framework 4.5.1 EOL platform), CVE-2021-45117 (autogenerated ANSI C
  stack stubs) — C/.NET-platform specific.
- CVE-2021-40142 (LDS DoS), CVE-2022-30551 / CVE-2023-32787 (Legacy Java Stack: block server
  from accepting requests) — the "exhaust the accept path" *class* is covered by our
  `hello_timeout` + connection caps above.
- CVE-2022-33916, CVE-2023-31048 (.NET *Reference Server* info leak) — implementation-specific
  to that sample; our sample/demo server should be spot-checked but no shared code path.

## Follow-ups

1. **Basic128Rsa15** stays deprecated and out of the default endpoint set — keep it that way;
   the pure-Rust RSA path's Marvin caveat is the only reason it matters. (See sad-rsa eval.)
2. **GCVE-105-2025-002** — spec-level advisory pending publication; monitor and re-review.
